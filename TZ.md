# Техническое задание (ТЗ) на разработку ПО "SteelFrame3D"

## 1. Общие сведения
- **Наименование системы:** SteelFrame3D
- **Тип системы:** Расчетно-аналитический программный комплекс для инженерных расчетов стальных каркасных конструкций методом конечных элементов (МКЭ).
- **Платформа:** Desktop (Windows, Linux, macOS).
- **Язык разработки:** Python 3.9+.
- **Лицензия:** (Указывается позже, например, MIT или Proprietary).

## 2. Назначение и цели создания
### 2.1. Назначение
Программа предназначена для статического расчета пространственных стержневых систем (ферм, рам, связей) из стали. Основные функции: моделирование геометрии, задание свойств материалов и сечений, приложение нагрузок, расчет усилий и перемещений, визуализация результатов.

### 2.2. Цели
- Предоставить инженерам бесплатный/доступный инструмент для предварительного расчета конструкций.
- Обеспечить прозрачность алгоритмов расчета (Open Core).
- Реализовать современный интерфейс на базе PyQt6/VTK.

## 3. Требования к функциональности

### 3.1. Моделирование геометрии
- **Создание узлов:** Ввод координат (X, Y, Z) вручную.
- **Создание стержней:** Соединение двух узлов. Автоматическое вычисление длины и направляющих косинусов.
- **Редактирование:** Перемещение узлов, копирование узлов, удаление элементов, отмена/повтор действий (Undo/Redo).
- **Назначение шарниров:** Назначение шарниров стержням.
- **Ориентация сечений:** Задание вектора ориентации `(Vx, Vy, Vz)` для каждого стержня для определения положения главных осей инерции сечения.

### 3.2. Свойства материалов и сечений
- **Библиотека материалов:** Сталь (C245, C255, C345 и др.) с заданными модулями упругости (E), сдвига (G) и плотностью.
- **Библиотека сечений:** Двутавры, швеллеры, трубы, уголки. Ввод геометрических характеристик (A, Iy, Iz, J, W).
- **Присвоение:** Возможность назначать материалы и сечения группам элементов.

### 3.3. Граничные условия и нагрузки
- **Закрепления:** Установка ограничений степеней свободы (UX, UY, UZ, RX, RY, RZ) в узлах.
- **Загружения:** Создание именованных схем загружений (Постоянное, Полезное, Снеговое и т.д.).
- **Типы нагрузок:**
  - Сосредоточенные силы и моменты в узлах.
  - Распределенные нагрузки на стержни (в локальной и глобальной СК).
- **Комбинации загружений:** Табличный редактор для создания линейных комбинаций с коэффициентами надежности.

### 3.4. Расчетное ядро
- **Метод:** Прямой метод перемещений (Direct Stiffness Method).
- **Элемент:** Пространственный стержень (6 степеней свободы в узле).
- **Решатель:** Решение системы линейных алгебраических уравнений (СЛАУ) $K \cdot U = F$.
- **Валидация:** Проверка матрицы жесткости на вырожденность (анализ кинематической неизменяемости) перед основным расчетом.

### 3.5. Визуализация и результаты
- **3D Вьювер:** Отображение схемы, деформированной схемы (с коэффициентом масштабирования).
- **Эпюры:** Отрисовка эпюр продольных сил (N), изгибающих моментов (My, Mz), поперечных сил (Qy, Qz).
- **Визуализация нагрузок:** Стрелочные маркеры (Gizmos), длина и цвет которых зависят от величины нагрузки.
- **Таблицы:** Вывод результатов в табличном виде с возможностью экспорта (CSV).

### 3.6. Работа с файлами
- **Формат:** Сохранение проекта в формате JSON (`.sf3d`).
- **Структура:** Версионирование, разделение данных на узлы, элементы, нагрузки и результаты.
- **Импорт/Экспорт:** (Перспектива) DXF, CSV.
	
## 4. Требования к надежности и безопасности
- **Целостность данных:** Автосохранение резервной копии при критических ошибках.
- **Обработка ошибок:** Программа не должна "падать" при некорректном вводе данных; должны выводиться понятные сообщения об ошибках.
- **Верификация:** Результаты расчета базовых задач должны совпадать с аналитическими решениями с погрешностью не более 1%.
	
## 5. Этапы разработки (Roadmap)

### Этап 1: MVP (Минимально жизнеспособный продукт)
- Ядро: Сборка глобальной матрицы жесткости, решение СЛАУ.
- Модель: Узлы, Стержни, Материалы, Сечения.
- Ориентация: Вектор `(Vx, Vy, Vz)`.
- Нагрузки: Сосредоточенные силы в узлах, Распределенные нагрузки на стержни.
- Комбинации: Простой табличный редактор.
- Визуализация: 3D схема, стрелочки нагрузок, деформированная схема.
- Интерфейс: Дерево проекта, панель свойств.
- Результаты: Эпюры усилий.
- Файл: Сохранение/Загрузка JSON.

### Этап 2: Расширение функционала
- Валидация: Базовая проверка (дубли узлов, отсутствие закреплений).

### Этап 3: Продвинутый анализ
- Валидация: Анализ кинематической неизменяемости (пробный прогон).
- Комбинации: Автогенерация по нормам (СП).
- Оптимизация: Подбор сечений.

### Этап 4: Релиз и дистрибуция
- Упаковка: Standalone installer (PyInstaller).
- Документация: Полное руководство пользователя.
- Тесты: Сверка с промышленным ПО (LIRA/SCAD).
