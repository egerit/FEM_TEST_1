## 1. Введение

### 1.1. Назначение документа
Настоящий документ описывает архитектуру, функциональные требования, структуру данных и пользовательский интерфейс программного комплекса **SteelFrame3D (SF3D)**. Программа предназначена для выполнения строительных расчетов металлических каркасов промышленных зданий методом конечных элементов (МКЭ) с использованием расчетного ядра OpenSeesPy.

### 1.2. Область применения
*   Расчет однопролетных и многопролетных промышленных зданий.
*   Моделирование конструкций с крановыми нагрузками.
*   Анализ статических, динамических и нелинейных режимов работы конструкций.
*   Некоммерческое использование (научные исследования, учебные цели, внутренняя экспертиза).

### 1.3. Цели и задачи
*   Создание модульного ПО с открытой архитектурой для расширения функционала.
*   Обеспечение полноценного 3D-моделирования без импорта из CAD.
*   Реализация всех типов анализа, доступных в OpenSees (статика, динамика, устойчивость, нелинейность).
*   Предоставление интуитивного интерфейса на базе PyQt6 с продвинутым 3D-вьювером (PyVista).
*   Экспорт результатов и исходных данных в формат Excel для верификации.

### 1.4. Глоссарий
*   **DOF (Degrees of Freedom):** Степени свободы узла (3 перемещения, 3 поворота в 3D).
*   **Release:** Освобождение степени свободы (моделирование шарниров).
*   **Load Case:** Загружение (набор нагрузок, действующих одновременно).
*   **Combination:** Комбинация загружений (линейная сумма результатов загружений с коэффициентами).
*   **OpenSeesPy:** Python-интерфейс к вычислительному ядру OpenSees.
*   **ЛСК:** Локальная система координат стержня.

### 1.5. Системы единиц
*   **Базовая система:** СИ (SI).
    *   Длина: метры [м].
    *   Сила: килоньютоны [кН].
    *   Напряжение: килопаскали [кПа] (или МПа, где удобно, с учетом масштаба).
    *   Масса: тонны [т] или кг (для динамических расчетов).
*   **Политика:** Все внутренние расчеты ведутся строго в базовой системе. Конвертация выполняется только на уровне ввода/вывода данных в UI (в перспективе).

## 2. Архитектура системы

### 2.1. Общая схема архитектуры
Приложение строится по паттерну **MVVM (Model-View-ViewModel)** или модифицированному **MVC**, обеспечивая строгое разделение логики данных, интерфейса и вычислений.

**Основные модули:**
1.  **Core (Ядро данных):** Хранение модели (узлы, элементы, нагрузки), бизнес-логика управления состоянием.
2.  **Solver Interface (Интерфейс решателя):** Преобразование внутренней модели в команды OpenSeesPy, запуск расчета, парсинг результатов.
3.  **UI (Пользовательский интерфейс):** Окна, диалоги, панели инструментов (PyQt6).
4.  **Visualization (Визуализация):** 3D-отрисовка сцены (PyVista/VTK), внедренная в Qt.
5.  **Database (БД профилей):** Работа с Excel-файлами сортаментов.
6.  **Exporter (Экспортер):** Генерация отчетов в Excel.

### 2.2. Технологический стек
*   **Язык:** Python 3.9 или выше.
*   **GUI Framework:** PyQt6.
*   **3D Engine:** PyVista (обертка над VTK) для встраивания в PyQt6 (`QVTKRenderWindowInteractor`).
*   **Calculation Core:** OpenSeesPy (`openseespy`).
*   **Data Handling:** pandas (табличные данные), openpyxl (Excel), numpy (матрицы).
*   **Serialization:** JSON со схемой валидации.
*   **Logging:** стандартный модуль `logging`.

### 2.3. Принципы взаимодействия модулей
*   **Событийная модель:** Изменения в модели генерируют события, которые обновляют UI и Вьювер.
*   **Асинхронность расчета:** Тяжелые вычисления запускаются в отдельном потоке (`QThread`), чтобы не блокировать интерфейс.
*   **Независимость ядра:** Модуль расчета не знает о GUI, он работает только с абстрактной структурой данных.

### 2.4. Управление данными (Data Flow)
1.  Пользователь вводит данные через **UI** -> Обновляется **Модель**.
2.  Пользователь запускает расчет -> **Solver Interface** читает **Модель** -> Генерирует скрипт OpenSees -> Выполняет расчет.
3.  **Solver Interface** получает результаты -> Сохраняет их в **Модель** (в объект `Results`).
4.  Событие "Расчет завершен" триггерит обновление **Вьювера** (эпюры) и доступность кнопок экспорта.

### 2.5. Формат файла проекта (Serialization)
*   **Формат:** Чистый JSON с явной схемой (JSON Schema).
*   **Структура файла:**
    ```json
    {
      "version": "1.0",
      "metadata": { "name": "...", "author": "...", "date": "..." },
      "units": { "length": "m", "force": "kN" },
      "materials": [ ... ],
      "sections": [ ... ],
      "nodes": [ ... ],
      "elements": [ ... ],
      "boundary_conditions": [ ... ],
      "load_cases": [ ... ],
      "combinations": [ ... ]
    }
    ```
*   **Преимущества:** Читаемость человеком, независимость от версий Python, возможность внешней валидации.
*   **Версионирование:** Поле `version` обязательно. При загрузке проверяется совместимость версии файла с версией программы.

## 3. Модуль данных и модели (Core)

### 3.1. Структура данных расчетной схемы
Основной класс `StructureModel` содержит коллекции объектов:

*   **Nodes (Узлы):**
    *   `id`: уникальный идентификатор.
    *   `x, y, z`: координаты.
    *   `constraints`: закрепленные степени свободы (фиксированные граничные условия).
*   **Elements (Стержни/Элементы):**
    *   `id`: уникальный идентификатор.
    *   `node_start`, `node_end`: ссылки на узлы.
    *   `section_id`: ссылка на сечение.
    *   `material_id`: ссылка на материал.
    *   `releases_start`, `releases_end`: маски освобожденных степеней свободы (для шарниров).
    *   `orientation_vector`: вектор ориентации местного сечения `[Vx, Vy, Vz]` в глобальной СК. Определяет направление оси Y локальной системы сечения (ось слабой жесткости для двутавра). По умолчанию `[0, 0, 1]` (вертикально вверх).
*   **Materials (Материалы):**
    *   Тип: `Elastic`, `Steel01` (билайн), `Steel02` (Гиафф-Пинто).
    *   Параметры: `E` (модуль упругости), `nu` (коэф. Пуассона), `fy` (предел текучести), `rho` (плотность).
*   **Sections (Сечения):**
    *   Тип: `WideFlange` (двутавр), `Channel` (швеллер), `Angle` (уголок), `Pipe` (труба), `Box` (коробка).
    *   Геометрические характеристики: `A`, `Iy`, `Iz`, `J`, `Cy`, `Cz` и др.
    *   Ссылка на запись в БД ГОСТ.
*   **Loads (Нагрузки):**
    *   Привязка к объекту (Узел, Элемент, Вся система).
    *   Тип: `PointLoad` (сила/момент в узле), `BeamLoad` (распределенная нагрузка на стержень), `MassLoad` (масса).
    *   Вектор нагрузки в глобальной или местной СК.
*   **AnalysisSettings (Настройки анализа):**
    *   Тип анализа (Static, Modal, Transient).
    *   Параметры нелинейности (шаги, толерантность).

### 3.2. База данных сечений
*   **Источник:** Файлы Excel (`GOST_Profiles.xlsx`) в папке приложения.
*   **Структура файла:** Листы по типам проката (Двутавры, Швеллеры, Трубы). Столбцы: обозначение, высота, ширина, толщина, площадь, моменты инерции и т.д.
*   **Функционал:**
    *   Загрузка при старте программы.
    *   Поиск и фильтрация.
    *   Возможность пользователю добавить свой лист или файл в ту же папку, программа подхватит его автоматически.
    *   Кэширование в памяти для быстрого доступа.

### 3.3. Модель материалов
По умолчанию используется сталь С245, С345, С375 (параметры по СП).
Пользователь может создавать пользовательские материалы с заданными свойствами.
Автоматический расчет массы стержня: $m = \rho \cdot A \cdot L$.

### 3.4. Система ограничений связей (Releases)
Для моделирования шарниров и полужестких соединений используется механизм "End Releases".
*   Реализуется через установку нулевой жесткости или специальных элементов `zeroLength` в местах сопряжения, либо через параметр `end_offsets` и модификацию матрицы жесткости элемента (в зависимости от возможностей элемента OpenSees).
*   В интерфейсе представляется как чекбоксы для 6 степеней свободы (UX, UY, UZ, RX, RY, RZ) для начала и конца стержня.

### 3.5. Комбинации загружений (Load Combinations)
*   **Модель данных:** Таблица комбинаций, где каждая строка содержит:
    *   `name`: Имя комбинации (например, "Основная 1").
    *   `factors`: Список пар `{load_case_id: coefficient}`.
    *   Пример: `{"LC1_Dead": 1.1, "LC2_Live": 1.4, "LC3_Wind": 0.9}`.
*   **Интерфейс:** Табличный редактор (QTableView).
    *   Столбцы: [Название комбинации, Загружение, Коэффициент].
    *   Возможность добавлять/удалять строки, редактировать коэффициенты.
    *   Группировка по имени комбинации.
*   **Расчет:** Результаты комбинации вычисляются постпроцессором как линейная сумма результатов загружений: $R_{comb} = \sum (R_{case\_i} \cdot k_i)$.
*   **Перспектива:** Автоматическая генерация комбинаций на основе выбранных норм (СП 20.13330) с классификацией нагрузок по типам (постоянные, временные длительные/кратковременные).

## 4. Модуль расчетного ядра (Solver Interface)

### 4.1. Генерация скрипта OpenSees
Класс `OpenSeesGenerator` проходит по объектам `StructureModel` и формирует последовательность команд:
1.  `wipe()` - очистка предыдущей модели.
2.  `model('BasicBuilder', '-ndm', 3, '-ndf', 6)` - создание 3D модели.
3.  Цикл по узлам: `node(...)`, `fix(...)`.
4.  Цикл по материалам: `uniaxialMaterial(...)`.
5.  Цикл по сечениям: `section('Elastic', ...)`.
6.  Цикл по элементам: `element('elasticBeamColumn', ..., '-release', ...)` или `nonlinearBeamColumn`.
7.  Цикл по нагрузкам: `load(...)`, `eleLoad(...)`.

### 4.2. Типы элементов
*   **Линейный расчет:** `elasticBeamColumn`.
*   **Геометрическая нелинейность:** `elasticBeamColumn` с флагом `-geomTransf PDelta` или `Corotational`.
*   **Физическая нелинейность:** `dispBeamColumn` или `forceBeamColumn` с интеграцией по длине и нелинейным сечением (`FiberSection`).
*   **Контакты:** Элементы `zeroLength` с нелинейными материалами (Gap, Hook) для моделирования зазоров и односторонних связей.

### 4.3. Типы анализа
Реализуются через стратегию `AnalysisStrategy`:
1.  **Статический линейный:** `system BandSPD`, `numberer RCM`, `constraints Plain`, `test NormDispIncr`, `algorithm Newton`, `integrator LoadControl`, `analysis Static`.
2.  **Геометрическая нелинейность (P-Delta):** Тот же алгоритм, но с геометрической трансформацией.
3.  **Физическая нелинейность:** Использование итерационного процесса Ньютона-Рафсона с малыми шагами загрузки.
4.  **Модальный анализ:** `analysis Modal`, получение собственных частот и форм колебаний.
5.  **Динамический (Transient):**
    *   **Сейсмика:** Чтение акселерограммы из файла (txt/csv), задание `GroundMotion`, использование `MultipleSupport` или `UniformExcitation`. Интегратор `Newmark`.
    *   **Гармонический:** Нагрузка вида $F(t) = F_0 \sin(\omega t)$.

### 4.4. Обработка результатов
После расчета выполняются команды `recorder Node -file ... -node ... -dof ... disp/reaction` и `recorder Element -file ... force/stress`.
Класс `ResultParser` считывает временные файлы OpenSees, преобразует их в `pandas.DataFrame` и сохраняет в объект `StructureModel.results`.
Структура результатов:
*   `nodal_displacements`: [NodeID, Step, DOF, Value]
*   `nodal_reactions`: [NodeID, Step, DOF, Value]
*   `element_forces`: [ElemID, Step, Location (N, Qy, Qz, Mx, My, Mz)]
*   `modal_periods`: [Mode, Period, Frequency]

### 4.5. Валидация модели (Pre-check)
Перед запуском расчета выполняется автоматическая проверка модели на корректность:
*   **Геометрическая целостность:**
    *   Проверка на дубликаты узлов (совпадающие координаты).
    *   Проверка на нулевую длину стержней.
*   **Статическая определимость/неизменяемость:**
    *   Проверка наличия достаточного количества закреплений (отсутствие механизмов).
    *   Пробный запуск анализа с единичной нагрузкой для проверки матрицы жесткости на вырожденность.
*   **Корректность данных:**
    *   Наличие назначенных материалов и сечений у всех элементов.
    *   Отсутствие отрицательных или нулевых характеристик сечений.
*   **Реакция на ошибки:** При обнаружении проблем расчет блокируется, пользователю выводится список ошибок с указанием ID проблемных объектов.
*   **Перспектива:** Интерактивная подсветка проблемных зон во вьювере (красным цветом выделяются узлы без закреплений, "висячие" элементы и т.д.).

## 5. Пользовательский интерфейс (UI)

### 5.1. Главное окно (MainWindow)
Компоновка типа "IDE":
*   **Верх:** Главное меню (Файл, Правка, Вид, Расчет, Сервис, Помощь) и Панель инструментов (Быстрые кнопки).
*   **Лево:** Панель **"Дерево проекта"** (QTreeView).
*   **Центр:** Область **3D Вьювера**.
*   **Право:** Панель **"Свойства элемента"** (QFormLayout / QTableWidget).
*   **Низ:** Панель **"Журнал/Log"** и статусная строка.

### 5.2. 3D Вьювер (PyVistaQt)
*   **Интеграция:** Виджет `QVTKRenderWindowInteractor` внутри PyQt6.
*   **Проекция:** По умолчанию ортогональная (`parallel_projection = True`), переключаемая на перспективную.
*   **Навигация:** Вращение, панорамирование, зум (стандартные жесты VTK).
*   **Отображение модели (режимы):**
    *   *Каркас (Wireframe):* Узлы – кубы (размер зависит от масштаба), Стержни – линии (толщина фиксирована или пропорциональна площади).
    *   *Сечения (Solid):* При включении режима "Отображать сечения", линии заменяются на 3D-геометрию профилей (extrusion профиля вдоль оси стержня). Переключение через чекбокс или горячую клавишу.
    *   *Шарниры:* Визуализируются как полупрозрачные сферы в концах стержней, где есть освобождения степеней свободы.
*   **Визуализация результатов:**
    *   Деформированная схема (с коэффициентом масштабирования).
    *   Цветовые эпюры (Clouds) на стержнях: усилия (M, N, Q), напряжения, перемещения.
    *   Векторы реакций в опорах.
    *   Анимация модальных форм колебаний.
*   **Визуализация нагрузок:**
    *   **Стрелочные маркеры (Gizmos):** 3D-стрелки в точках приложения сосредоточенных сил и моментов.
    *   Длина и толщина стрелки пропорциональны величине нагрузки.
    *   Для распределенных нагрузок – цепочка стрелок вдоль стержня или сплошная цветовая лента.
    *   Переключение видимости нагрузок по типам и загружениям.
*   **Выбор объектов:**
    *   Ray-picking (клик мышью) выделяет узел или стержень.
    *   Выделенный объект подсвечивается цветом.
    *   Синхронизация выделения с "Деревом проекта" и "Панелью свойств".

### 5.3. Панель "Дерево проекта"
Иерархическая структура:
*   📂 **Геометрия**
    *   📄 Узлы (список с номерами)
    *   📄 Стержни (список с номерами)
*   📂 **Свойства**
    *   📄 Материалы
    *   📄 Сечения
*   📂 **Нагрузки**
    *   📁 Загружение 1 (Dead Load)
        *   📄 Силы в узлах
        *   📄 Нагрузки на стержни
    *   📁 Загружение 2 (Live Load)
*   📂 **Граничные условия**
*   📂 **Результаты** (появляется после расчета)
    *   📄 Перемещения
    *   📄 Усилия
    *   📄 Реакции

*Действия:* Двойной клик открывает редактор. ПКМ (правая кнопка) вызывает контекстное меню (Добавить, Удалить, Переименовать, Скрыть/Показать).

### 5.4. Панель "Свойства элемента"
Контекстно-зависимая панель. Отображает поля для редактирования выбранного объекта.
*   Если выбран **Узел**: Координаты (X, Y, Z), Закрепления (чекбоксы DX, DY, DZ, RX, RY, RZ).
*   Если выбран **Стержень**: Начальный/Конечный узел, Материал (выпадающий список), Сечение (выпадающий список), Ориентация (вектор или угол), Освобождения (чекбоксы Start/End для 6 DOF).
*   Если выбрана **Нагрузка**: Значения сил/моментов, тип распределения, направление.
*   *Функционал:* Изменения вступают в силу сразу (или по кнопке "Применить"), модель обновляется, вьювер перерисовывается.

### 5.5. Редактор таблиц загружений
Отдельное диалоговое окно или вкладка для массового ввода нагрузок.
*   Таблица (QTableView + Pandas Model): Столбцы [ID, Тип, Элемент/Узел, Значение, Направление, Координата начала, Координата конца].
*   Фильтрация по типу нагрузки.
*   Импорт/Экспорт таблиц в CSV/Excel.

### 5.6. Панель управления расчетом
*   Выбор типа анализа из списка.
*   Кнопка "Запустить расчет".
*   Прогресс-бар (если возможно оценить длительность).
*   Кнопка "Стоп".
*   Окно вывода логов OpenSees (ошибки сходимости, предупреждения).

## 6. Модуль отчетности и экспорта

### 6.1. Формирование таблиц
Использование библиотеки `pandas` для структурирования данных.
Таблицы должны содержать:
*   Номер узла/элемента.
*   Номер загружения/комбинации.
*   Значения усилий/перемещений в характерных точках (начало, 1/4, 1/2, 3/4, конец).
*   Максимальные значения для элемента.

### 6.2. Экспорт в Excel
Библиотека `openpyxl` или `xlsxwriter`.
Структура файла отчета:
1.  **Лист "Info":** Дата, версия ПО, имя файла проекта.
2.  **Лист "Input Data":** Дамп всей расчетной схемы (координаты, свойства сечений, материалы, нагрузки) для верификации входных данных.
3.  **Лист "Displacements":** Перемещения узлов.
4.  **Лист "Reactions":** Реакции в опорах.
5.  **Лист "Forces":** Усилия в стержнях (разбито по компонентам).
6.  **Лист "Modal":** Периоды и частоты собственных колебаний.

Каждый лист снабжается фильтрами автофильтрации Excel.

## 9. Риски и ограничения
*   **Производительность:** Отрисовка тысяч 3D-сечений в PyVista может быть ресурсоемкой. Требуется реализация LOD (Level of Detail) - показывать сечения только при зуме.
*   **Сходимость нелинейных задач:** OpenSees чувствителен к параметрам решения. Потребуется удобный интерфейс тонкой настройки солвера.
*   **Лицензирование:** OpenSees имеет открытую лицензию, но необходимо соблюдать требования при распространении.
