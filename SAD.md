# Архитектурный документ (SAD) SteelFrame3D

## 1. Введение
### 1.1. Назначение документа
Документ описывает высокоуровневую архитектуру программного комплекса SteelFrame3D, определяет основные компоненты системы, их взаимодействие и технологический стек.

### 1.2. Область применения
Архитектура предназначена для разработки десктопного приложения на Python, реализующего метод конечных элементов (МКЭ) для расчета стержневых систем с графическим интерфейсом.

## 2. Архитектурные принципы
- **Разделение ответственности (SoC):** Четкое разделение между логикой расчета (Core), данными (Model) и интерфейсом (UI).
- **Независимость от UI:** Расчетное ядро не должно зависеть от библиотек интерфейса (PyQt/VTK). Ядро должно работать как standalone библиотека.
- **Data-Driven:** Состояние приложения полностью описывается моделью данных. UI является лишь представлением этой модели.
- **Расширяемость:** Возможность добавления новых типов элементов, материалов и нагрузок без переписывания ядра.

## 3. Общая архитектура системы
Система строится по паттерну **Model-View-Presenter (MVP)** или модифицированному **MVVM**, где:
- **Model:** Данные конструкции и математические модели.
- **View:** Визуализация (VTK) и виджеты управления (PyQt).
- **Presenter/Controller:** Логика приложения, связывающая действия пользователя с изменениями в модели и пересчетом.

### 3.1. Диаграмма компонентов (логическая)

```text
+-----------------------+       +-----------------------+
|      User Interface   | <---->|    Application Core   |
| (PyQt6 + VTK Widgets) |       |   (Controller Logic)  |
+-----------------------+       +-----------------------+
                                         |
                  +----------------------+----------------------+
                  |                                             |
          +---------------+                           +------------------+
          | Domain Model  |                           |  Calculation     |
          | (Data Classes)|                           |  Engine (Core)   |
          +---------------+                           +------------------+
                  |                                             |
          +---------------+                           +------------------+
          | Serialization |                           |   Math Solvers   |
          | (JSON Handler)|                           | (NumPy/SciPy)    |
          +---------------+                           +------------------+
```

## 4. Описание компонентов

### 4.1. Уровень представления (Presentation Layer)
**Технологии:** `PyQt6`, `vtkmodules.qt.QVTKRenderWindowInteractor`.

- **MainWindow:** Главное окно приложения, содержащее меню, тулбары и центральную область.
- **VTKViewerWidget:** Виджет для 3D-отрисовки.
  - Отвечает за рендеринг геометрии (линии, точки).
  - Отрисовка Gizmos (стрелки нагрузок, оси координат).
  - Визуализация эпюр (трубчатые или ленточные геометрии поверх стержней).
  - Камера и навигация.
- **PropertyEditor:** Панель свойств выбранного объекта (узла, стержня, нагрузки).
- **ModelTree:** Дерево проекта (Загружения, Комбинации, Группы элементов).
- **TableViews:** Таблицы для ввода координат, редактирования комбинаций.

### 4.2. Уровень приложения (Application Layer)
**Технологии:** Pure Python.

- **ProjectManager:** Управление жизненным циклом проекта (New, Open, Save, Close).
  - Работает с файловой системой.
  - Контролирует версию файла.
- **CommandProcessor:** Реализация паттерна Command для поддержки Undo/Redo.
  - `AddNodeCommand`, `DeleteElementCommand`, `ModifyLoadCommand`.
- **SelectionManager:** Управление выделением объектов в сцене и синхронизация с таблицами.
- **ValidationService:** Запуск проверок модели перед расчетом.

### 4.3. Уровень предметной области (Domain Layer)
**Технологии:** Pure Python (Dataclasses / Pydantic).

- **StructureModel:** Корневой объект модели.
  - `nodes`: List[Node]
  - `elements`: List[BeamElement]
  - `materials`: Dict[str, Material]
  - `sections`: Dict[str, Section]
  - `load_cases`: Dict[str, LoadCase]
  - `combinations`: List[LoadCombination]
- **Node:** ID, X, Y, Z, BoundaryConditions (mask).
- **BeamElement:** ID, StartNodeID, EndNodeID, MaterialID, SectionID, OrientationVector.
- **Load:** Тип (Force/Moment), Значения (Fx, Fy, Fz...), Привязка (NodeID или ElementID + param).
- **LoadCombination:** Name, List[FactorPair(LoadCaseID, Coefficient)].

### 4.4. Расчетное ядро (Calculation Core)
**Технологии:** `numpy`, `scipy.sparse`, `scipy.sparse.linalg`.

- **MatrixAssembler:** Сборка глобальной матрицы жесткости $K$.
  - Вычисление локальных матриц жесткости для каждого стержня.
  - Поворот в глобальную СК (используя OrientationVector).
  - Ассемблирование в разреженную матрицу (CSR format).
- **BoundaryConditionApplier:** Модификация $K$ и $F$ согласно закреплениям (метод штрафных функций или исключение строк/столбцов).
- **Solver:** Решение $K \cdot U = F$.
  - Использование `spsolve` для разреженных систем.
- **ResultExtractor:** Вычисление усилий в стержнях по найденным перемещениям.
  - Возвращает N, Qy, Qz, My, Mz, T для каждого элемента.

### 4.5. Уровень доступа к данным (Data Access Layer)
**Технологии:** `json`, `schema`.

- **JsonSerializer:** Преобразование `StructureModel` <-> JSON dict.
- **SchemaValidator:** Проверка структуры JSON на соответствие схеме (версия, обязательные поля).

## 6. Ключевые сценарии взаимодействия (Flow)

### 6.1. Сценарий расчета
1. Пользователь нажимает кнопку "Рассчитать".
2. `CommandProcessor` проверяет валидность модели через `ValidationService`.
3. Если ошибок нет, `ProjectManager` передает текущую `StructureModel` в `CalculationCore`.
4. `MatrixAssembler` строит матрицу $K$ и вектор $F$ для каждой комбинации.
5. `Solver` находит вектор перемещений $U$.
6. `ResultExtractor` вычисляет усилия.
7. Результаты сохраняются в объекте модели (в поле `results`).
8. UI обновляется: включается доступ к просмотру эпюр.

### 6.2. Сценарий сохранения
1. Пользователь выбирает "Сохранить как...".
2. `JsonSerializer` преобразует модель в словарь.
3. Словарь проверяется по JSON Schema.
4. Данные записываются в файл `.sf3d` (UTF-8).

## 7. Требования к производительности
- **Время сборки матрицы:** Для модели до 10,000 стержней — не более 2 секунд.
- **Время решения:** Для модели до 10,000 стержней — не более 5 секунд (на среднем ПК).
- **Отзывчивость UI:** Интерфейс не должен зависать во время расчета (использование `QThread` или `asyncio` для тяжелых вычислений).

## 8. Риски архитектуры
- **GIL (Global Interpreter Lock):** Может ограничивать многопоточность в чистом Python.
  - *Митигация:* Тяжелые вычисления вынесены в NumPy/SciPy, которые освобождают GIL.
- **Сложность интеграции VTK и PyQt:** Возможны конфликты событий мыши.
  - *Митигация:* Использование готовых виджетов `QVTKRenderWindowInteractor` и тщательное тестирование.
